# PRINCE2 Stage Control Pack

Use this workflow during an active delivery stage when the project manager needs to control progress and keep the project within tolerance.

## Inputs To Ask For

- Current stage plan
- Active work packages
- Latest checkpoints
- Risks, issues, changes, and dependencies
- Tolerances for time, cost, quality, scope, benefits, risk, and sustainability
- Decisions or escalations needed

## Output Structure

### 1. Stage Control Snapshot
- Stage objective
- Current status
- Tolerance position
- Confidence level
- Immediate management action

### 2. Work Package Control
- Work package
- Owner or team manager
- Status
- Products due
- Quality checks due
- Risks or blockers
- Action needed

### 3. Highlight Report
- Period covered
- Progress this period
- Planned next period
- Current and forecast tolerance position
- Risks and issues
- Decisions required

### 4. Risk, Issue, And Change Updates
- New entries
- Updated impact
- Owner
- Response
- Escalation threshold
- Link to affected plan or product

### 5. Corrective Actions
- Action
- Owner
- Due date
- Expected effect
- Follow-up point

## Quality Bar

- Distinguish normal variance from a tolerance threat.
- Escalate exceptions early.
- Keep work package tracking tied to product delivery and quality criteria.
- Update logs and registers as part of the output, not as an afterthought.
