# Project + Product Skills

Use this skill kit when the user is doing day-to-day project or product management work.

## How to choose a skill

- Match the user's request to one specific skill when possible.
- Prefer operating artifacts over generic advice: briefs, plans, checklists, decision notes, scorecards, and runbooks.
- For PRINCE2 requests, map "phases" to PRINCE2 processes and use `prince2-documentation-map` when the user asks which management products are required.
- If the request spans multiple workflows, sequence the work instead of blending every framework into one oversized response.

## Default style

- Spell out product management instead of using shorthand.
- Use direct language and concrete next actions.
- Surface assumptions, risks, owners, dates, and decisions.
- Avoid filler, vague strategy language, and decorative formatting.
