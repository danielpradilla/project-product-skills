# PRINCE2 Documentation Map Example

Use `prince2-documentation-map` before creating a pile of templates. It helps decide which management products are needed and which can be combined.

Prompt:

```text
Use prince2-documentation-map for a medium-risk internal software migration.

Context:
- 4-month project
- one internal engineering team
- data migration risk
- sponsor wants monthly board updates
- no external supplier
- benefits will be measured after go-live
```

Expected output:

- Tailored documentation strategy
- Process-to-documentation map
- Core management products to create
- Products that can live in delivery tools
- Products that can be combined or skipped
